SHERPA: A Knowledge Acquisition Mediator

  • Goal: support maintaining KBSs

  • Problem: many KBSs are not built with our framework

  • Proposed solution: Knowledge Acquisition Mediator

    • routes information to many applications from one source, i.e., a knowledge library
    • uses an Interdependency Model of how knowledge is used in an application

  • Benefits:

    • selective notification to application builders of library updates
    • library maintainers can evaluate the impact of a change
    • tools to acquire factual knowledge from domain experts can be built using the model
    • model can be used to describe the application to someone who is considering using the application
