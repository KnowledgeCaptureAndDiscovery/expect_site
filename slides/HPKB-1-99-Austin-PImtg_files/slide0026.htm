EXPECT
Expectation-Based Method Development Tool (EMeD)
nEffective KA tools guide users with expectations derived from dependencies among components in KBS
–User: Add Havana as a new seaport
–System: Tell me about the berths of Havana because I need to know how to dock ships
nProblem addressed:During initial KB creation when there is a smaller body of knowledge in the KB, where can we get expectations from?
nApproach: Discovered several sources of expectations for KB creation
äGuide the design and creation of a new KB element (e.g., a method)
–Find dependencies within the KB element based on representation language (ex: variables must be defined, tests must return boolean values)
–Find if the new method fits in principle with existing knowledge
äDetect missing knowledge
–Find undefined methods  given the newly created ones
–Propose initial version of new methods to add (e.g., propose a result type)
äFitting pieces together
–Find user defined and yet unused methods
–Propose potential uses of an unused method in other methods
