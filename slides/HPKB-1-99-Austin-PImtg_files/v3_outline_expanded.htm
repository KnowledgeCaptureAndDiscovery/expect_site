EXPECT’s Approach to COA Critiquing:
A Reusable Method for Plan Evaluation
Approach: Design and reuse of a PSM that captures general principles, in terms of
Plan ontology: represents manually or automatically created plans, tasks, goals, commitments, causal links, etc.
Critiquing and evaluation ontology (output ontology): “a critique is the result of analyzing a feature of the plan and determining whether it deviates from a given norm”
Evaluation PSMs, e.g., “a goal statement that is missing any of its subcomponents is incomplete”
Current scope of PSM:
Analysis of the plan structure (e.g., “check missing task decomposition links”)
Analysis of resource use (e.g., check that non-shareable resources are not used by different tasks that overlap in time)

Expectation-Based Method Development Tool (EMeD)
Effective KA tools guide users with expectations derived from dependencies among components in KBS
User: Add Havana as a new seaport
System: Tell me about the berths of Havana because I need to know how to dock ships
Problem addressed:During initial KB creation when there is a smaller body of knowledge in the KB, where can we get expectations from?
Approach: Discovered several sources of expectations for KB creation
Guide the design and creation of a new KB element (e.g., a method)
Find dependencies within the KB element based on representation language (ex: variables must be defined, tests must return boolean values)
Find if the new method fits in principle with existing knowledge
Detect missing knowledge
Find undefined methods  given the newly created ones
Propose initial version of new methods to add (e.g., propose a result type)
Fitting pieces together
Find user defined and yet unused methods
Propose potential uses of an unused method in other methods

EXPECT’s Support for KB Modifications:
Knowledge Acquisition Scripts
Problem addressed:  KA tool supports wide range of KB modifications, but support provided could improve
tool considers KB errors in isolation
tool does not point out which problems to attend first
suggested fixes are often too many and too general
Approach: KA Scripts capture prototypical sequences of changes & are used to guide users complete modifications
support the user in completing a “transaction”
Research issues: designing a KA Script library, coordinating the application of KA Scripts, user interaction
Benefits: preliminary evaluations with users show:
modifications take less time, especially complex ones
KA Scripts act as a checklist that reminds users of changes

Script-Based Knowledge Acquisition:
Encovering and Using KA Strategies
Designed a library of KA modification strategies
Current implementation: 35 strategies (soon to be 100)
Estimated size: 300 strategies
Initial evaluation of script-based KA tool:
Method development time reduced 28%-36%
Recreation of Workarounds modification phase
Method creation rate: 6.3 min/method
Total KB extension: 17 methods, 108 min
ALL new methods created and modified using KA Scripts
