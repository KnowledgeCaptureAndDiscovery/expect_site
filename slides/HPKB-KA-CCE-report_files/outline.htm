Background
Subjects extended the KB of the EXPECT COA Critiquer
The EXPECT COA critiquer provides an analysis of COA structure and resource use
Functionality of our tools: help users enter problem solving fragments
“In order to estimate the force ration for a task in a COA, compare the combat power of all Red units performing the task with the combat power of all the Red units involved”
Knowledge acquired: EXPECT’s fine-grained problem-solving methods, which contain procedural constructs such as conditional statements and iteration)
EXPECT’s Approach to Knowledge Acquisition
Key idea:  Tool can understand how different pieces of knowledge are related, and guides user to provide knowledge needed to make them work together
       EXPECT forms expectations about what knowledge it needs to acquire
EXPECT exploits many sources of expectations:
Existing domain knowledge
General principles and middle-level theories
How problem solving works
Typical acquisition strategies

Snapshot of EMeD interface
An Example of EMeD’s Interaction with Users
User adds a new method
“In order to evaluate the force-ratio for a military task, check if the required ratio for the task is less than the available ratio for the task”.
System points out that it is missing problem solving knowledge
EMeD has analyzed interdependencies between the new method and its sub-methods (for computing required ratio and for computing available ratio) and found that the user has to add a new method for computing available ratio.
System proposes an initial sketch of missing knowledge
EMeD predicts what the missing sub-method has to do and proposes a capability : the new method should compute available ratio for a given military task
EMeD predicts the output of the missing sub-methods, and proposes a result type : the new method should return a number (force ratio) to be compared with another number (required force ratio)

Experiments with the EXPECT PSM-Based KA Scripts and NL Editor (Jim Blythe)
These experiments test a KA tool that uses:
PSM-based KA Scripts   [Blythe & Gil 99] that use  background knowledge about plan evaluation and critiquing (a Problem-Solving Method + ontologies) to guide the dialog with a user about a new critique to be added
An English-based editor  [Blythe & Ramachandran 99] that lets the user add or modify problem-solving knowledge by manipulating its English paraphrase, without having to see or understand the internal formal representation
PSM-Based KA Scripts
Helps the user add new critiques by using a background theory (PSM) of plan evaluation and critiquing.
KA Scripts guide the dialog with the user about the new critique (wizard-type interaction).
The tool creates some of the needed methods for the critiques, helps the user to create new ones (by suggesting initial templates), and ensures consistency with existing knowledge.
English-Based Editor
Generates automatically English paraphrases of
problem-solving fragments, and presents alternative

the paraphrase based
on the ontologies and

Experiment with PSM-Based KA Scripts
Summary
Using the PSM-Based KA Scripts significantly reduced the time taken to add a critique
Using the PSM-Based KA Scripts, all four subjects successfully added simple critiques to the EXPECT critiquer; three of them successfully added more complex critiques.
Without the PSM-Based KA Scripts, three out of four subjects successfully added simple critiques and two added more complex critiques.
Comments on the tool usability were positive in all cases.
Second experiment with PSM-Based KA Scripts and English-Based Editor
Hypothesis: the combination of the PSM-Based KA Scripts and English-based editor allows a user with very little training to add new critiques.
Single subject usability test: A subject was briefed in COA critiquer and the KA interface (but not about EXPECT) for 20min and asked to add two critiques using the tool
KA tasks: add two new critiques to the EXPECT COA critiquer (a completeness check and a resource check), same used in the previous experiment
Subject: an Army officer with no EXPECT training
Results
The subject was able to add two new critiques of both low and medium complexity.

Golden Nuggets

Lumps of coal:
Overall time savings is not a good measure
Tool only helps with a subset of all the KA activities
Individual differences are important (and hard!) to control
Conclusions
Domain experts were able to use our KA tools to enter new COA critiques and adapt existing ones
Our experiments show that
our KA techniques (interdependency models, PSM-based KA scripts, English-based editor) broaden the range of knowledge that domain experts can enter
problem-solving knowledge (including procedural constructs)
our KA techniques help users enter knowledge faster
A subject with practically no training was able to add new COA critiques
KA tool uses background knowledge, NL-based interface, and acquisition scripts
We are developing a good methodology to evaluate KA tools
