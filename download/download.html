Download, Install and Setup directions for the EXPECT system.

System Requirements:

Unix X environment (any window manager)
Allegro Common Lisp with CLIM (tested on ACL 4.3 and CLIM 2.0)
LOOM (version 3.0)

Download :

Loom can be got from its official homepage here
Expect system can be obtained from here
A brief tutorial can be found here and the manual here.

Installation :

a) Download and install LOOM3.0 according to the instructions that are found on the LOOM homepage.
 
b) After downloading EXPECT.tgz extract it to the location that you want to run it from
note that it will create a top-level directory structure called expect.
 
c) The Directory structure that can be expected to be seen under EXPECT is as follows:
drwxrwxrwx BIN <-- Main directory for the compiled files
-rwxrwxrwx compile.lisp <-- file for compiling EXPECT
-rwxrwxrwx defsys.lisp <-- load file list (see defsystem.text)
-rwxrwxrwx defsystem.lisp
-rwxrwxrwx defsystem.text
drwxrwxrwx documentation <-- limited EXPECT documentation
drwxrwxrwx domains <- Some practice domains for EXPECT
drwxrwxrwx grator
drwxrwxrwx ka
drwxrwxrwx kas
-rwxrwxrwx load-path-redefs.lisp
-rwxrwxrwx load.lisp <-- file to load EXPECT
drwxrwxrwx matcher <- EXPECT matcher files
-rwxrwxrwx pathnames.lisp
drwxrwxrwx ps <-- EXPECT problem solving files
drwxrwxrwx shared <-- EXPECT shared utilities and functions
drwxrwxrwx tests <-- Test cases for the EXPECT system
drwxrwxrwx ui <-- All UI files for the EXPECT system

Setup and running the EXPECT system :

a) The LOOM architecture can be loaded directly from the lisp files but this will degrade system performance greatly. It is strongly recommended to compile LOOM first. To do this, after loading your version of acl with CLIM, load the compile-loom.lisp file.
 
b) The EXPECT architecture can be loaded directly from the lisp files but this will degrade system performance greatly. It is strongly recommended to compile EXPECT first. To do this, after loading acl, CLIM and LOOM, load the compile.lisp file.
 
c) If needed the EXPECT image may be saved at this time for an easy load for the next time by (dumplisp :name <filename> :standalone t) or the equivalent in your version of ACL.
 
d) The EXPECT UI can be started from the Lisp prompt by forse switching to the expect package
USER> :pa expect
and then typing EXPECT>(e)

Practice Domains for EXPECT:

These currently include :
a) The factorial domain (calculate the factorial of a number)
b) The Propose and revise algorithm
c) The Uhaul configuration domain (based on the propose-and-revise algorithm)
d) The transportation domain (for the calculation of time required to transport)
In addition to these, more complex domains such as COA Critiquing, INSPECT, etc. will be available on request.
